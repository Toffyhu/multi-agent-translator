# 系统架构详解 — WorkBuddy 翻译流水线

> 本文档详细描述翻译流水线的架构设计、Agent分工、数据流和关键技术决策。

---

## 1. 总体架构

### 1.1 流水线流程

```
┌────────────────────────────────────────────────────────────────────────┐
│                        翻译流水线执行流程                                │
└────────────────────────────────────────────────────────────────────────┘

源文本（英文）
     │
     ▼
┌─────────────────────────────────────┐
│   Stage 1: Book Analyzer            │
│   - 全书内容分析                      │
│   - 文学风格评估 + 关键角色提取        │
│   - 输出: 分析报告（JSON）            │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│   Stage 2: Term Stylist             │
│   - 基于分析报告构建术语表            │
│   - 设定风格参数                     │
│   - 输出: 术语表 + 风格指南（JSON）   │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│   Stage 3: Chapter Translator       │
│   ← 逐章处理                        │
│   ← 加载术语表+风格指南              │
│   ← 调用Chief Editor[每章]           │
│   - 输出: 中文译文（Markdown）        │
└──────────────────┬──────────────────┘
                   │
                   ▼
┌─────────────────────────────────────┐
│   Stage 4: Chief Editor             │
│   循环执行直至通过质量门：            │
│   ┌──────────────────────┐          │
│   │ Editor → 审校+修改    │          │
│   │ ↓ 评分 < 7?          │          │
│   │ 是 → 返回Translator   │          │
│   │ 否 → 通过             │          │
│   └──────────────────────┘          │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│   Stage 5: Fact Checker             │
│   - 专有名词准确性验证               │
│   - 历史事实核对                     │
│   - 输出: 终稿（Markdown）           │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│   Stage 6: Post-Process + Delivery  │
│   - 清理 CHANGES/TERMS 标记         │
│   - 修复重复章节标题 + 统一格式      │
│   - PDF 生成（WeasyPrint）           │
│   - 邮件发送（可选）                  │
└─────────────────────────────────────┘
```

### 1.2 Quality Gate 机制

每章翻译输出后，Chief Editor 进行质量评分（1-10分）：

| 分数 | 含义 | 行动 |
|------|------|------|
| 1-4 | 严重质量问题 | 返回Translator重译 |
| 5-6 | 需要大量修改 | Editor直接修改 |
| 7-8 | 少量润色 | 快速修正后放行 |
| 9-10 | 通过 | 直接交付 |

## 2. Agent 详细设计

### 2.1 BaseAgent（所有Agent的基类）

```python
class BaseAgent:
    - 统一LLM调用接口
    - 自动注入共享知识库（SharedKnowledge）
    - 多模型切换支持
    - 错误重试 + 上下文截断
```

**自动知识注入逻辑**（在 `_call_llm()` 中实现）：

```python
knowledge_block = self.assets.get_knowledge_prompt_block()
if knowledge_block and len(knowledge_block) > 50:
    system_prompt = f"""{system_prompt}

## 📚 共享翻译技能知识（所有Agent必须遵循）
{knowledge_block}"""
```

### 2.2 Book Analyzer

| 属性 | 值 |
|------|-----|
| 类名 | `BookAnalyzerAgent` |
| 模型 | Qwen3.7-Max（分析任务最复杂） |
| 输入 | 全书原文文本 |
| 输出 | JSON: 风格评估 + 角色关系 + 核心主题 + 章节摘要 |

### 2.3 Term Stylist

| 属性 | 值 |
|------|-----|
| 类名 | `TermStylistAgent` |
| 模型 | Qwen3.6-Flash（最简单任务，性价比最高） |
| 输入 | Book Analyzer的分析报告 |
| 输出 | JSON: 术语表（原文→译文）+ 风格参数（词汇等级/句式偏好） |

### 2.4 Chapter Translator

| 属性 | 值 |
|------|-----|
| 类名 | `ChapterTranslatorAgent` |
| 模型 | Qwen3.7-Plus（默认）/ Qwen3.7-Max（文学类） |
| 输入 | 章节原文 + 术语表 + 风格指南 |
| 输出 | Markdown 译文（纯文学导向，无技术规则） |

**提示词核心**（完整版见 `src/models/prompts/chapter_translator.md`）：

> "你是一位专业的英→中文学翻译。你的翻译风格应该富有文学性、充满创造力、让中文读者沉浸其中。你不是在'转换文字'，你是在用中文**重述**这个故事。"

### 2.5 Chief Editor

| 属性 | 值 |
|------|-----|
| 类名 | `ChiefEditorAgent` |
| 模型 | Qwen3.7-Plus |
| 输入 | 译文原文对照 |
| 输出 | 修正后的译文 + 修改记录 + 质量评分 |

**检查范围**（举一反三）：
1. 代词指代不清
2. 中英标点混用
3. 语序不符合中文习惯
4. 逻辑链断裂（需回溯英文原文）
5. 用词不当/翻译腔
6. 段落结构缺陷

### 2.6 Fact Checker

| 属性 | 值 |
|------|-----|
| 类名 | `FactCheckerAgent` |
| 模型 | Qwen3.7-Plus |
| 输入 | 终稿译文 |
| 输出 | 事实修正报告 + 最终译文 |

### 2.7 Content Scout

| 属性 | 值 |
|------|-----|
| 类名 | `ContentScoutAgent` |
| 模型 | 无需LLM（纯API调用） |
| 输入 | 搜索关键词 |
| 输出 | JSON: 中译空白作品列表（含价值评分） |

**中译检测算法**（三重验证）：
1. **豆瓣搜索**：解析 `/subject/\d+/` 链接数量
2. **Bing站点限定**：`site:douban.com/subject {title}`
3. **保守web搜索**：仅当多个查询都返回正向信号才判定为有中译

## 3. 模型选型策略

```yaml
# 精打细算策略（见 config/models.yaml）
book_analyzer:    aliyun/qwen3.7-max      # 最贵 → 分析任务最复杂
term_stylist:     aliyun/qwen3.6-flash    # 最便宜 → 术语任务最简单
chapter_translator:
  default:        aliyun/qwen3.7-plus     # 中间 → 日常翻译
  literary:       aliyun/qwen3.7-max      # 最贵 → 高文学性作品

fact_checker:     aliyun/qwen3.7-plus     # 中间
```

| Qwen3.6-Flash | 免费 | 无限 | 术语等简单任务 |

## 4. 共享知识库

`shared_knowledge_base.md` 包含7个章节、约2809字符的翻译方法论，自动注入到所有Agent的system prompt中：

| 章节 | 内容 |
|------|------|
| 1. 翻译方法论 | 策略→方法→技巧三层结构 |
| 2. 核心原则 | 代词/标点/段落/诗歌处理 |
| 3. 中英差异 | 重心/结构/语态对比 |
| 4. 表达优化 | 中文表达提升技巧 |
| 5. 风格策略 | 按文学类型分类处理 |
| 6. 文化适应 | 文化负载词的处理策略 |
| 7. 质量清单 | 自检清单 |

来源：Tencent IMA 知识库（翻译理论专家 × 翻译世界）+ 熊兵等学术论文。

## 5. 后处理流水线

```python
def post_process_delivery(raw_text: str) -> str:
    1. 清理 ---CHANGES--- / ---TERMS--- 编辑标记
    2. 检测并修复重复/空章节标题
    3. 统一中英标点（英文引号→中文引号）
    4. 合并多余空行
    5. 验证章节完整性
    6. 返回干净文本
```

## 6. 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| Agent框架 | Hermes Agent v0.16+ | 轻量、Pythonic、支持多Agent编排 |
| 翻译模型 | Aliyun Qwen 系列 | 有免费额度、质量可接受、国内直连 |
| 知识库注入 | 自动注入 system prompt | 所有Agent无需额外调用即可访问知识 |
| 质量评估 | 1-10分定量评分 | 可程序化判断是否需重译 |
| PDF生成 | WeasyPrint + Noto CJK | 开源、支持完整CSS |
| 邮件发送 | QQ SMTP 465端口 | 国内稳定、可程序化控制 |
| 上下文截断 | 段落边界感知 | 避免在句中截断破坏语义 |
